Before creating a new quest you should:
• Read this guide to know what is important for a new quest
• Have a look at this list where all quests are listed

If you need help, don't hesitate to contact me: info.ent8r@gmail.com

General options

  • Kind of the quest
    The kind of the quest can either be a simple Yes-No quest where the user can only answer with Yes or No or a quest with images for each answer (used by the parking type quest).
  • Importance of the quest
    The importance of a quest specifies when the quest will be downloaded and displayed. High values will be displayed and downloaded earlier, low values later.

The Yes-NoQuest

  • Directory
    A short, descriptive name for the quest you want to implement. Should be lowercase, contain no numbers and underscores instead of spaces.

    e.g. "building_levels" or "fire_hydrant"
  • Question
    The question which should be asked to the user.

    e.g. "What is the speed limit sign for this street?"
  • Overpass query
    Probably the most complex part: the overpass query. This specifies which kind of objects should be found and displayed to the user. If you need help you can ask on Github or have a look at the wiki page where you can learn how to write overpass queries.

    e.g. "nodes with highway=crossing and !crossing"
  • OSM tag which should be modified
    A specific key which is used by the OpenStreetMap and other projects to identify an object.

    e.g. "amenity", "lit" or "building"
  • Commit message
    The commit message which describes the changes which were made.

    e.g. "Add housenumbers" or "Add speed limits"

Optional answers for the Yes-No-Quest

  • OSM value for the answer "Yes"
    The OSM value which should be set for the answer "Yes". By default this is "yes".
  • OSM value for the answer "No"
    The OSM value which should be set for the answer "No". By default this is "no".

The image quest

  • OSM value for the answer
    The OSM value which should be set for this answer.

    e.g. "amenity", "lit" or "building"
  • Answer
    An answer for the question. It is only required if you think that it will help the user to find the correct answer. (If the image is not very self-descriptive)
  • The image
    The image which helps the user to find an answer. The image has to be quadratic and must have a resolution higher than 384*384. And it is also very important that the image has a fitting license (CC0, CC-BY-SA). You can find good images at the wiki of OSM, wikipedia or pixabay

Optional answers for the image quest

  • Images per row
    Determines how much images should be displayed in a row. By default is this 4.
  • Initially shown answers
    The number of initially shown items determines how many images should be displayed at the first view. By default are all images displayed at once.
  • Other answer
    With this option you can specify other answers which are not directly visible at the first view. But after clicking on "Other answer" the user can select this answer.
